# 串口显示协议（Host → ESP32）

电脑上的 host 程序通过 USB 串口把数据发给 ESP32，ESP32 在 128×32 OLED 上显示
Claude / Codex 的额度以及中转站余额。

**设计原则：固件只负责「渲染」，不含任何切换/翻页/选择逻辑。**
host 每发一行 JSON 就是「一整屏」(Frame)，固件收到就照着画。要显示哪一页、什么时候翻页、
数据是否过期，全部由 host 决定——这样 host 可以灵活控制显示，固件保持极简。

固件解析实现见 [firmware/src/main.cpp](firmware/src/main.cpp)。

---

## 1. 传输方式

| 项目 | 值 |
|------|----|
| 接口 | USB-CDC 串口（C3/S3 一根 USB 线即可） |
| 波特率 | **115200** |
| 格式 | **NDJSON**：一行一个 Frame（JSON 对象），以 `\n`（换行）结尾 |
| 编码 | ASCII（`\r` 会被忽略） |
| 单行长度 | < 600 字节（超长行会被丢弃） |

> 每发一行就整屏刷新一次。非法 JSON、空行（<2 字符）会被静默忽略。
> 翻页由 host 主动驱动：host 每隔约 4 秒发下一页（`both` 显示对象时在多页间轮播）。
> 固件不保存历史页、不自己翻页；它永远只显示「最近收到的那一帧」。
> 兜底：超过 **90 秒**没收到任何新行，固件会本地标 `old`（数据过期）。

---

## 2. Frame：一整屏

每一帧用 `k` 字段区分两种页型：

| `k` | 含义 | 用途 |
|-----|------|------|
| `"pct"`   | 百分比页 | Claude / Codex 官方订阅的 5 小时 + 1 周剩余额度 |
| `"money"` | 金额页   | 中转站的「已用 / 今日用量」+ 总余额 |

通用字段：

| 字段 | 类型 | 默认 | 说明 |
|------|------|------|------|
| `ic` | string | — | 图标。`pct` 页：`"claude"` / `"codex"`（内置位图）；`money` 页：货币符号（画在硬币上，如 `"$"`） |
| `k`  | string | `"pct"` | 页型，见上表 |
| `l1` | string | — | 第 1 行标签（最长 ~13 字符） |
| `l2` | string | — | 第 2 行标签 |
| `old`| bool   | `false` | 数据过期 → 固件叠加显示 `old` |

---

## 3. `pct` 百分比页

表示两个窗口的**剩余百分比**（上行 5 小时、下行 1 周）。

| 字段 | 类型 | 默认 | 说明 |
|------|------|------|------|
| `p1` | int | `-1` | 第 1 行剩余百分比 0–100；`-1` → 显示 `ERR` |
| `s1` | string | `""` | 第 1 行重置时间文本，最长 11 字符（如 `15:47`） |
| `p2` | int | `-1` | 第 2 行剩余百分比；`-1` → `ERR` |
| `s2` | string | `""` | 第 2 行重置时间（如 `06-25 14:30`） |

显示效果（左侧 provider 图标 + 右侧两行：标签 + 进度条 + 大字百分比 + 重置时间）：

```
[icon] 5h  ▆▆▆▆░░  94%
       15:47
       1w  ▆▆░░░░  38%
       06-25 14:30
```

示例：

```json
{"ic":"claude","k":"pct","l1":"5h","p1":94,"s1":"15:47","l2":"1w","p2":38,"s2":"06-25 14:30"}
```

出错 / 无数据时（任一行 `p=-1` 即显示 `ERR`）：

```json
{"ic":"codex","k":"pct","l1":"5h","p1":-1,"l2":"1w","p2":-1}
```

---

## 4. `money` 金额页（中转站）

显示中转站的用量与总余额。第 1 行的标签由 host 决定：

- **sub2api** 中转站：展示「今日用量」，标签发 `"Today"`。
- **new-api** 中转站：展示「累计已用」，标签发 `"Used"`。

| 字段 | 类型 | 默认 | 说明 |
|------|------|------|------|
| `l1` | string | `"Used"` | 第 1 行标签（`Today` / `Used`） |
| `v1` | string | `"--"` | 第 1 行金额文本（**已含货币符号**，如 `$1.2`） |
| `l2` | string | `"Left"` | 第 2 行标签 |
| `v2` | string | `"--"` | 第 2 行金额（总余额，如 `$880`） |

> 金额文本由 host 直接格式化好（整数不带小数，非整数保留 1 位），固件原样显示。
> 货币符号既写在 `v1`/`v2` 里，也通过 `ic` 单独给出（画在左侧硬币上）。

显示效果（左侧货币硬币 + 右侧两行大字）：

```
[ $ ]  Today   $1.2
       Left    $880
```

示例：

```json
{"ic":"$","k":"money","l1":"Today","v1":"$1.2","l2":"Left","v2":"$880"}
```

---

## 5. host 如何驱动显示

固件没有「显示对象 / 翻页」概念；这些都在 host 侧实现（见 `host/engine.go` 的 `displayLoop`）：

| host 的「显示对象」 | host 轮播的页序列 |
|----|----|
| `claude` | 只发 Claude `pct` 页 |
| `codex`（官方） | 只发 Codex `pct` 页 |
| `codex`（中转） | 只发 `money` 页 |
| `both`（官方 codex） | Claude `pct` → Codex `pct`，每 ~4s 翻一页 |
| `both`（中转 codex） | Claude `pct` → `money`，每 ~4s 翻一页 |

host 每 ~4 秒发当前页；取到新数据时也会立刻重发当前页。固件只是被动跟随。

---

## 6. 字段长度限制（固件缓冲区）

| 字段 | 最大长度 |
|------|---------|
| `ic` | 9 字符 |
| `l1` / `l2` / `s1` / `s2` | 13 字符 |
| `v1` / `v2` | 15 字符 |
| 整行 JSON | 600 字节 |

超出会被截断，整行超 600 字节会被丢弃。

## 7. 本地测试

不依赖 host 程序，可用脚本直接发测试帧：
[firmware/tools/send-test.ps1](firmware/tools/send-test.ps1)

```powershell
# firmware 目录下执行
powershell -ExecutionPolicy Bypass -File tools\send-test.ps1 -Kind pct   -Port COM3
powershell -ExecutionPolicy Bypass -File tools\send-test.ps1 -Kind money -V1 "$1.2" -V2 "$880"
```

> 脚本独占串口，运行时不能同时开 host 程序。端口号用 `pio device list` 查。
